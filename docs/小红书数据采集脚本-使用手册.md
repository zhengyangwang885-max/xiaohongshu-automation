# 小红书数据采集脚本 · 使用手册

> 本手册记录了在本机搭好的小红书采集环境（基于开源工具 MediaCrawler）的安装、配置与用法。
> 换电脑或重装时照此复现即可。

---

## 一、一句话用法

```bash
# 按关键词搜笔记（最常用：找选题 / 看赛道热度）
xhs-crawler --type search --keywords "AI工具,AI副业" --crawler_max_notes_count 20
```

跑完后，数据自动存成 Excel：`~/MediaCrawler/data/xhs/xhs_search_日期.xlsx`

---

## 二、三种采集模式

| 模式 | 命令 | 用途 |
|---|---|---|
| 搜索 search | `xhs-crawler --type search --keywords "关键词1,关键词2"` | 按关键词批量抓笔记，找选题/看热度 |
| 账号 creator | `xhs-crawler --type creator --creator_id "主页URL或ID"` | 扒某个对标账号的全部笔记 |
| 详情 detail | `xhs-crawler --type detail --specified_id "笔记URL1,笔记URL2"` | 抓指定笔记的详情+评论 |

### 常用附加参数

| 参数 | 说明 | 示例 |
|---|---|---|
| `--keywords` | 关键词，逗号分隔 | `--keywords "AI工具,AI教程"` |
| `--crawler_max_notes_count` | 最多抓多少篇 | `--crawler_max_notes_count 20` |
| `--get_comment` | 是否抓评论（yes/no） | `--get_comment yes` |
| `--lt` | 登录方式 qrcode/phone/cookie | `--lt qrcode` |
| `--creator_id` | 对标账号 ID 或主页链接 | `--creator_id "https://..."` |
| `--specified_id` | 指定笔记 ID 或链接 | `--specified_id "https://..."` |

> 说明：`xhs-crawler` 是封装命令，等价于
> `cd ~/MediaCrawler && ./.venv/bin/python main.py --platform xhs ...`

## 三、采集到的字段（Excel 列）

note_id、type（图文/视频）、title、desc、video_url、time、user_id、nickname、
liked_count（赞）、collected_count（藏）、comment_count（评）、share_count、

## 四、登录说明

1. 首次运行会弹出浏览器，显示小红书登录二维码 → 用手机小红书 App 扫码。
2. 可能出现滑块/点选验证码 → 需手动完成。
3. 登录态会保存（`SAVE_LOGIN_STATE = True`），之后再采集一般不用重复扫码。
4. 建议用**小号**登录采集，降低主号风险。

## 五、环境复现步骤（换机/重装时用）

```bash
# 1. 安装 uv（依赖管理 + 装独立 Python）
curl -LsSf https://astral.sh/uv/install.sh | sh
export PATH="$HOME/.local/bin:$PATH"

# 2. 克隆仓库
cd ~ && git clone --depth 1 https://github.com/NanmiCoder/MediaCrawler.git
cd ~/MediaCrawler

# 3. 建 Python 3.12 虚拟环境并装依赖（用 requirements.txt，避开旧 Pillow 编译）
uv venv --python 3.12
uv pip install -r requirements.txt

# 4. 关键修复：签名库降级到兼容版本（否则搜索报 sign_state 错误）
uv pip install "xhshow==0.1.9"

# 5. 安装浏览器内核
./.venv/bin/playwright install chromium

# 6. 改两个配置（见下方"六、关键配置"）

# 7. 建全局命令 xhs-crawler（见下方"七、启动脚本"）
```

## 六、关键配置（`~/MediaCrawler/config/base_config.py`）

| 配置项 | 改成 | 原因 |
|---|---|---|
| `ENABLE_CDP_MODE` | `False` | 默认 True 会去连本机 Chrome 调试端口 9222，没开就一直等。改 False 用内置浏览器 |
| `SAVE_DATA_OPTION` | `"excel"` | 默认 jsonl，改 excel 方便做对标分析 |
| `HEADLESS` | `False`（默认即是） | 扫码登录需要看得见浏览器窗口 |

## 七、启动脚本 `~/.local/bin/xhs-crawler`

```bash
#!/bin/bash
# MediaCrawler 小红书对标采集 一键封装
cd "$HOME/MediaCrawler" || exit 1
exec ./.venv/bin/python main.py --platform xhs "$@"
```

创建后执行 `chmod +x ~/.local/bin/xhs-crawler` 赋予执行权限。

## 八、常见问题

| 现象 | 原因 / 解决 |
|---|---|
| 一直 `CDP port 9222 is not accessible` | `ENABLE_CDP_MODE` 没改成 False |
| 搜索报 `got multiple values for argument 'sign_state'` | xhshow 版本太新，降级到 0.1.9 |
| 编译 Pillow 报缺 jpeg | 别用 `uv sync`（锁旧 Pillow），改用 `uv pip install -r requirements.txt` |
| 扫码窗口没出现 / 超时 | 重新运行；确认 HEADLESS=False |
| 抓不到数据 / 被风控 | 降低频率、减少 max_notes_count、换小号、隔段时间再抓 |

## 九、合规与风险提示

- MediaCrawler 是第三方开源爬虫，使用可能违反小红书用户协议。
- 只采集**公开数据**，不要高频、不要刷量、不要拿去做侵权或商业转售。
- 账号有被限流/封禁风险，**建议用小号**登录采集。
- 数据仅用于自己的选题研究与对标分析。
